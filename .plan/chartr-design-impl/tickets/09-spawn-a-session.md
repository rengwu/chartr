# Spawn a session

## Question

The product's tracer bullet. From a frontier ticket's pane (or the sidebar's affordances), spawn wires the whole chain: the chartr writes the claim commit — pathspec-limited to the ticket file, carrying trailers for layer provenance and the payload content hash (ADR 0008) — composes the payload, writes it to a gitignored path inside the space, archives it per session in chartr-owned state, resolves the role binding, and the adapter launches the agent's own interactive TUI in a PTY (no headless mode) with the one-line "read this file" opener typed in. The session seats as a tab bound to exactly one ticket. An absent binary hard-blocks that one spawn with the message naming the binding, its source layer, and the local-override fix. Which roles are offered follows the map's kind; an unclassified map offers none.

Done when: with a stub agent CLI on PATH, process-boundary tests assert the claim commit (pathspec, trailers), the gitignored payload whose content matches the preview, the archived copy, and the opener arriving at the stub's stdin; spawning from the pane in the browser lands on a live TUI tab; binding the role to a missing binary blocks with the specific message and blocks nothing else.

## Answer

The tracer bullet is wired end to end: one HTTP action turns a frontier ticket into a claim commit, an archived payload, and a live agent TUI in a PTY. It reuses the multiplexer and prompt machinery the earlier slices built and adds only the session seam on top. Layout:

- **`internal/adapter/adapter.go`** — the agent-agnostic seam (ADR 0002, as amended): an `Adapter` maps a binding's `{model, args}` onto one agent's argv (`claude`/`codex` take `--model`; an unrecognised name falls back to a generic `--model` adapter rather than refusing — the operator may bind any CLI). `Opener(path)` is the one line typed into the TUI: a plain "read this file in full … then begin" with a trailing newline, so injection is one path for every agent and no system-prompt flag is relied on.
- **`internal/terminal`** — the PTY layer already owned ad-hoc shells; it now also owns sessions. `newProc` is the one launch path both share; `OpenSession` runs the adapter's command instead of a shell, tags the tab with a `Session` (map slug, ticket, role, agent, model), types the opener in once the PTY is up, and enforces **one live session per space** (`ErrSessionExists`, `HasLiveSession`). A session tab is exactly an ad-hoc tab that additionally carries a `Session` — the only thing that tells the two apart — so liveness sampling, scrollback, attach/detach, and the pushed-tab plumbing are reused untouched. `model.Terminal` gains an optional `session` object on the wire.
- **`internal/server/claim.go`** — the chartr's one lifecycle write here (ADR 0008). `writeClaimCommit` stamps `claimed_by`/`claimed_at` into the ticket's frontmatter (so it derives `claimed`) and commits **only that file**: `git add -- <ticket>` then `git commit --only -- <ticket>`, a partial commit built from HEAD plus the one path, so whatever else sits staged in the operator's index can never be swept in. It handles a not-yet-tracked ticket and an unborn HEAD, never amends, never pushes. The message carries structured trailers — `Session`, `Agent`, `Model`, `Role`, `Payload-SHA256`, and the three `*-From` layer-provenance lines.
- **`internal/server/spawn.go` + route in `server.go`** — `POST /api/spaces/{id}/maps/{slug}/tickets/{num}/spawn {role}`. It discovers fresh off disk, overlays the committed kind, and gates: an **unclassified map offers no sessions** and a role the kind does not offer is refused (`config.RolesForKind`/`KindOffersRole` — planning grills/prototypes/researches, implementation implements/reviews); a non-frontier ticket is refused; **an absent bound agent hard-blocks that one spawn** with the resolver's own message naming the binding, its source layer, and the local-override fix. Only then does it compose the payload (the same `prompt.Compose` the preview uses), hash it, write the claim, drop the payload to the gitignored `.chartr/run/<id>/payload.md` (a `*` `.gitignore` beside it, re-asserted each spawn), archive a copy under `<dataDir>/sessions/<id>/`, and `OpenSession`. Every failure that can happen happens before launch, so a refused spawn leaves the space untouched save the claim, which by ADR 0008 stands once written.
- **frontend** (`model.ts` session type + `rolesForKind`, `actions.ts` `spawnSession`, a Spawn control on `DetailPane` threaded `SpacePane → MapCard → DetailPane`, and session-aware sidebar rows in `App.svelte`) — a frontier ticket on a classified map shows a role picker (the kind's roles) and a Spawn button; a block surfaces the chartr's specific message inline; the new session's tab is made active and renders in the sidebar bound to its ticket (`role #NN`, agent, status), told apart from a shell.

Against Done-when — five process-boundary tests in `internal/server/spawn_test.go`, all asserting on the HTTP response, the control-socket snapshot, the filesystem, and git history, driven against a stub agent CLI on PATH (new `chartrtest.StubAgent`, which records the agent's stdin): **(1)** the whole chain — claim commit touching exactly the ticket file with all trailers, the ticket deriving `claimed`, the gitignored payload byte-matching the preview and provably `git check-ignore`d, the archived copy, the opener arriving at the agent's stdin naming the payload, and a live session tab bound to one ticket (`TestSpawnWiresTheWholeChain`); **(2)** a missing agent blocking only that spawn — 409 with the specific message, no commit written, ticket still open, space still usable as a multiplexer (`TestSpawnMissingAgentBlocksOnlyThatSpawn`); **(3)** kind gating — unclassified offers none, a cross-lifecycle role refused (`TestSpawnRespectsKind`); **(4)** a held ticket refused (`TestSpawnRefusesNonFrontier`); **(5)** one session per space (`TestSpawnOneSessionPerSpace`). `go vet ./...`, `go test ./...`, `svelte-check` (0/0), `vitest` (33), and the Vite build all pass; no amber in the built CSS.

Three decisions for review to weigh:

- **A session is an ad-hoc tab plus a `Session`, not a new subsystem.** Rather than a parallel session manager, the terminal `Manager` grew `OpenSession`, and the wire `Terminal` grew an optional `session`. This reuses all of ticket 05's liveness/scrollback/attach plumbing and keeps "one tab list" in the sidebar, at the cost of `terminal.Terminal` no longer being *only* ad-hoc. Liveness sampling, the death-halt, resume/abandon, and quiet-vs-idle are deliberately **not** built here — that is tickets 10–12; a session today is a live PTY that ends when its process does or the human closes its tab.
- **The gitignored payload lives at `.chartr/run/<id>/payload.md`.** A sibling of the committed `config.toml`/`prompts/` under the same chartr directory, kept out of git by a `*` `.gitignore` the chartr writes into `run/` on every spawn (so the ignore holds on a fresh clone that never had the directory). This keeps everything the chartr writes into a space in one place; the alternative of a top-level ignored path was rejected as more litter in the operator's tree.
- **Spawn requires the ticket be on the stricter frontier.** A fresh spawn takes only an open, unclaimed, fully-unblocked ticket. Review-on-a-`proposed`-ticket and resume/respawn of a dead session are genuinely tickets 11 and 10, so the frontier gate is correct here but will need to widen when those land — flagged so review does not read it as the permanent rule. Role wiring travels in the payload body; the adapter passes `--model` and the binding's portable args and nothing else.

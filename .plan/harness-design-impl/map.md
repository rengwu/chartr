# wayfinder-harness — implementation

## Destination

The [spec](../harness-design/spec.md) implemented end to end: one distributed, self-contained binary an operator runs to register spaces, watch maps appear and derive live on the star-map, spawn prompt-injected agent sessions in real TUIs against frontier tickets, and gate implementation work through agent review and the human review hub — shipped as a checksummed release for macOS, Linux, and Windows. Done looks like the release ticket resolved with every slice before it demoable in the real cockpit.

## Notes

**This map carries execution.** Every ticket is a `task` that delivers working code, not a decision — all decisions were made on the [planning map](../harness-design/map.md) and synthesized into the [spec](../harness-design/spec.md), which is the single source of truth here. Do not re-litigate a decision; if implementation exposes one as wrong, mark the *planning* ticket undermined and raise it, rather than quietly deviating.

**Per-session reading order:** the spec, then this map, then your ticket — and any planning asset the ticket names (the UI tickets lean on the prototypes under `../harness-design/assets/`). Vocabulary comes from `CONTEXT.md` at the repo root; the ADRs in `docs/adr/` are binding.

**Testing:** tests live at the process boundary — start the real binary against a temp fixture space, act over HTTP, assert on control-socket snapshots, the filesystem, and git history, with stub agent CLIs on PATH (spec, Testing Decisions). Extend the rig the walking skeleton establishes; do not add internal seams. The star-map island's mount/model/selection seam is the one frontend test point.

**Before commit:** run the static checks and tests as wired by the walking skeleton (`go vet ./...` and `go test ./...`; the frontend's check and build scripts) — run what exists at your ticket's point in the map. Review the diff (the `review-code` skill if available). No map linter is wired in this repo.

## Decisions so far

<!-- one line per resolved ticket: gist + link. -->

- **01 — walking skeleton**: one Go binary serves the embedded Svelte cockpit shell and pushes the whole derived model over a JSON control socket (resent on reconnect); operator actions are plain HTTP; the process-boundary test rig every later ticket extends is established. [ticket](tickets/01-walking-skeleton.md)
- **02 — register a space, and role bindings**: the space registry is a rebuildable index in user config (register with an announced `git init`, forget-not-destroy removal, pinned-then-recency ordering, a path-derived stable id); role bindings resolve `{adapter, model, args?}` across built-in ‹ committed workspace (`.wayfinder-harness.toml`) ‹ local user, field-level and user-over-workspace, with absent adapters badged and a committed autopilot flag ignored with a warning; register/deregister/pin are plain HTTP actions that rebuild-and-push the model. [ticket](tickets/02-register-a-space-and-role-bindings.md)
- **03 — maps appear and derive**: the wayfinder-maps model layer is ported whole (ADR 0001) and extended by exactly the non-resolving `proposed` status (ADR 0004), so the ported `Frontier` is already the stricter one — a merely-proposed blocker never unblocks its dependents. Discovery is layout-agnostic (a map is found by its `map.md`, handling `.plan/<slug>/` and `.plan/maps/<slug>/`, hard-coding neither) and tolerant (a malformed map renders as-is with its malformation surfaced, never refused); an fsnotify watch over each space's `.plan/` makes maps appear by notice, not refresh; the sidebar nests spaces → maps with finished maps sorting last. [ticket](tickets/03-maps-appear-and-derive.md)
- **04 — classify a map's kind**: kind is declared, never inferred (ADR 0007) — a discovered map is inert (offers no session actions) until a human classifies it, with the `-impl`-suffix / all-`task` conventions surviving only as a pre-filled one-keystroke guess. Classification is one HTTP action writing a `[maps."<slug>"]` kind into committed workspace config (the shared layer, so teammates agree); the write *appends* rather than re-encodes, sparing the operator's own bindings and comments, and refuses an already-declared slug. An unrecognised committed kind is surfaced-and-inert; a renamed map directory dangles its entry back to unclassified-and-inert, never an error. The classify write lands in the working tree uncommitted — the harness's own commits stay the enumerated lifecycle writes (ADR 0008). [ticket](tickets/04-classify-a-maps-kind.md)
- **05 — an ad-hoc shell in the space**: the multiplexer baseline — a PTY layer on `aymanbagabas/go-pty` (cross-platform, ConPTY-capable from day one — ADR 0006 as amended) owns the harness's shells, so the session core never ossifies unix-only. An ad-hoc shell opens in a space's working tree over a plain HTTP action and streams over its own **binary terminal socket** (scrollback replayed on attach, raw bytes down, keystrokes up as binary frames, resize up as a text control message) to an xterm.js island the Svelte chrome hosts but never reaches inside (ADR 0010) — a separate connection from the control socket, so a flooding shell cannot block map updates. Ad-hoc shells are deliberately *not* sessions (no ticket, no lifecycle, ended by the human), so a mapless space is fully usable as a plain multiplexer; a slow socket is dropped to reattach-and-replay rather than back-pressuring the read loop. Terminals ride the pushed model as harness runtime state (never the raw bytes), so tabs appear and disappear by notice. [ticket](tickets/05-an-ad-hoc-shell-in-the-space.md)
- **06 — the star-map card**: the renderer reimplemented cleanly in TypeScript behind the narrow island seam — mount, receive the pushed model, emit selection — never decomposed into components (ADR 0010), with the tuned constants (easing, zoom coupling, parallax, dpr) cribbed from the wayfinder-maps renderer against feel-drift. Layout is a pure function of ticket numbers and `blocked_by` edges (never status) and seeded in `num` order, so it is deterministic and snapshot-order-independent; the renderer recomputes positions only when the map's *structure* changes, so a pure status push changes a star's appearance but never moves it. The six base states derive from `.plan/` status (ADR 0004), with `proposed` as the one added base star and the session/review grammar left to a later additive overlay. The map is summoned as a floating card over the terminal (edge handle, `M`, `Esc`) or docked as the terminal-priority split — resizable from its left border in both modes, the terminal holding its frozen width so a window resize never reflows it. The island degrades gracefully with no 2D context, which makes the seam (determinism, zero movement, selection) testable headless — the one frontend test point (vitest + jsdom). [ticket](tickets/06-the-star-map-card.md)
- **07 — the ticket pane**: selection opens a responsive detail pane over the island, assembled whole from the derived model with no second fetch — a ticket's Question and Done-when, its blockers with their answers pulled *inline* (`sectionOf` over each blocker's own Answer/Proposed Answer/Ruled out, degrading a missing one to a labelled note), and a session-history stub until the session tickets land; the map's title opens the map material the same way (Destination lifted, the duplicate heading stripped). The dock choice is a pure, tested function offering three strategies (width/aspect/hybrid) each with a dead-band so a drag through the boundary never flip-flops; the card runs hybrid — right by default (`min(400px, 58%)`), re-docking to the bottom (capped at half the panel) when it is narrow (< 600px) or portrait — and a ResizeObserver measures the pane so the camera eases the star into the space it leaves free in both dockings. Selection lives on the pane and rides a deep link (`#s&m&t`, or `&mat` for the material): parsed at boot to seat the linked star, re-applied on hashchange, reflected back via `replaceState` without looping. [ticket](tickets/07-the-ticket-pane.md)
- **09 — spawn a session**: the tracer bullet — one HTTP action (`POST …/tickets/{num}/spawn {role}`) turns a frontier ticket into a live agent TUI. A new `internal/adapter` maps a binding onto an agent's argv (agent-agnostic, `--model`; ADR 0002) and owns the read-this-file opener typed into the PTY; `internal/terminal` grew `OpenSession` so a session is an ad-hoc tab that additionally carries a `Session` (map/ticket/role/agent/model) — reusing all of ticket 05's liveness/scrollback plumbing, one live session per space. The claim is the harness's one write (ADR 0008): `claimed_by`/`claimed_at` stamped into frontmatter, committed *only that file* (`git commit --only`, handles untracked/unborn-HEAD, never amends/pushes) with agent/model/role/payload-hash/layer-provenance trailers. Gating: an unclassified map offers no sessions and a cross-lifecycle role is refused (`config.RolesForKind`); a non-frontier ticket is refused; an absent bound agent hard-blocks that one spawn with the specific naming-the-binding message and blocks nothing else. The composed payload (same `prompt.Compose` as the preview) is written gitignored to `.wayfinder-harness/run/<id>/` and archived under `<dataDir>/sessions/<id>/`. Liveness, the death-halt, and review are left to 10–12. [ticket](tickets/09-spawn-a-session.md)
- **08 — the prompt library and payload preview**: a new `internal/prompt` package owns the hackable library — `core` + five role prompts + `glossary`, embedded and materialized to `<dataDir>/prompts/*.md` as the editable base (edits compose on the next preview, never clobbered). Each part resolves across built-in ‹ user ‹ workspace with `replace` (resets base, discarding layers below) / `append` (stacks); the committed workspace layer wins over local user (content half of ADR 0009), and a `replace` recording `<!-- forked from <hash> -->` that no longer matches the shipped default is surfaced as behind — on both the preview and the space — never auto-merged. `Compose` assembles core + role + a fresh context bundle (glossary, map body, ticket, blockers' answers inline — ADR 0005); the **review** payload additionally carries the ticket's Done-when and the spec by assembly, the spec discovered from the map's own `](…spec.md)` link rather than a hard-coded path. A read-only `GET .../tickets/{num}/payload?role=` composes fresh off disk; a `PayloadPreview` modal renders the payload with per-segment layer provenance from the ticket pane. Prompts inject via the payload body, never an agent system-prompt flag. [ticket](tickets/08-the-prompt-library-and-payload-preview.md)
- **10 — liveness and the death halt**: a session that dies stays pinned to its ticket (scrollback intact) instead of dropping — the manager's exit callback pins a natural death and drops only an ad-hoc shell or an operator-killed tab (`pinOnDeath = session && !killed`). The halt offers exactly three HTTP choices and enacts none: **resume** (same-ticket crash recovery, ADR 0005 — same id, archived payload re-materialized, claim stands), **respawn** (fresh session, a new claim re-stamped over the stale one via an idempotent `stampClaim`), **release** (`writeReleaseCommit` strips the claim and commits only the ticket, back to `open`). The spawn mechanics are factored into `Server.launchSession`, shared by spawn and respawn. Liveness is surfaced, never acted on: the terminal samples output silence and reports a raw `Silent` verdict; the server turns it into **quiet** only for an AFK role (`config.RoleIsAFK` — `grill` is the sole HITL role) whose ticket carries no `## Proposed Answer` yet, past a `QuietAfter` threshold. A dirty tree (`git status --porcelain` → `Space.Dirty`) badges but never gates a spawn. Seven process-boundary tests assert the whole thing on the public seam (snapshot, filesystem, git), including that nothing changes without an operator call. [ticket](tickets/10-liveness-and-the-death-halt.md)
- **12 — the human review hub**: the gate as four plain HTTP actions in `internal/server/gate.go`, keyed on one choke point (`gateTarget`) that assembles the brief on demand from the reviewer's on-disk verdict — closing a hole where an unassembled verdict meant approve asked for one fewer tick, not one more. **Approve** promotes `## Proposed Answer` → `## Answer` and commits narrowly (`internal/server/promote.go`, `git commit --only`, never an amend), detecting the attribution smear an empty commit against a clean path implies (ADR 0008 — reported, not failed) and answering with the next best frontier ticket; over a rejecting verdict it demands `acknowledged` and records the override (`Approved-Over-Rejection`, `Acknowledged-Blocking`). **Send back**/**take it further** share one `follow-up` endpoint — ticket 09's `launchSession` with `prompt.Steer` riding the payload and its archive only, never the ticket. **Abandon** demotes the proposal to a dated `### Rejected` section, strips the claim back to `open`, and offers `revert`/`reset` as unticked levers. `model.Ticket.Review` is the explicit gate signal ticket 13 flagged needing. `web/src/lib/ReviewHub.svelte` takes over the map card, brief-first with the full verdict and diff behind expanders. Six process-boundary tests cover every exit; the agent review of this ticket found its one browser-facing Done-when clause unmet (the post-approve strip destroys itself on approve, plus five advisories) — resolved here on human approval over that verdict, with the fix tracked as its own unit, ticket 17, rather than folded back in. [ticket](tickets/12-the-human-review-hub.md)

- **17 — the review hub, driven by eye**: ticket 12's own fix-up — the blocking finding was two bugs stacked, not one: `MapCard.svelte`'s closing effect unmounted the whole hub the instant approve's rebuild pushed a resolved snapshot, and `ReviewHub.svelte`'s reload effect was keyed to prop *identity* rather than value, so it would have wiped `approved` on the very next unrelated push even with the hub kept mounted. `hubApproved` is lifted into `MapCard` (exempting the locally-approved ticket from the close effect) and the reload effect now keys off a computed string compared against its previous value, the same idiom `DetailPane.svelte` already used for the identical problem; the Next button now spawns the suggestion directly or falls back to selecting it. Backend: a per-ticket review pointer (`.wayfinder-harness/run/reviews/<slug>/<num>`) replaces the session-tab scan so the newest verdict wins and survives a discarded tab or a restart; every harness lifecycle commit now carries a `Harness-Write: true` trailer so work-commit matching stops guessing from the subject line; `scope=read` diffs surface a real error instead of a fake empty one; `demoteProposal` groups every abandonment under one `## Rejected attempts` heading; `resetAvailable` mirrors abandon's own tip check so the dialog's second lever only appears where the backend would accept it. Six new process-boundary tests. Flagged: the browser click-through itself did not run (no Chrome connection in this session) — approved anyway on the human driver's read, same precedent as 12; and `resetAvailable` is correct but rarely true through the hub as built, since reaching the exits section requires a review, and a review's own claim always breaks `tipOf`'s strict tip check — a pre-existing ticket-12 characteristic, not re-decided here. [ticket](tickets/17-the-review-hub-driven-by-eye.md)

- **13 — session states on the star-map**: the moons grammar as a strictly-additive overlay inside the island (ADR 0010) — a session is a body, an amber moon orbiting the star it holds. `starmap/session.ts` holds the vocabulary: `GRAMMAR` names each state's hue *and* its non-colour channels (motion, moon carriage, marks) and the renderer draws *from* it, so "nothing is colour-only" is a property of the paint, not a claim about it; `sessionStates(map, terminals)` derives all six states per push from the snapshot alone (tickets + session tabs), storing nothing. Liveness is motion (orbits with a trail / crawls dimmed and blinking / freezes grey with its orbit greyed), the pipeline is position (docked at the rim once proposed, with agent review's violet counter-orbiter — the one new hue), and human review breaks the orbital grammar on purpose (whitened dock, gold star, pings, and a gold edge chevron when it scrolls offscreen). A live change flares its star once and writes one fading ticker line, both painted by the island. `setModel` took the overlay as a second argument rather than a second seam call, so one push is one visual beat; the overlay is never a layout input, so ticket 06's zero-movement guarantee holds through the whole lifecycle. Flagged: `human-review` is derived as "a `proposed` ticket whose review session has exited" until ticket 12 offers an explicit signal. [ticket](tickets/13-session-states-on-the-star-map.md)

## Out of scope

<!-- Inherited from the spec's Out of Scope; these never graduate into tickets on this map. -->

- **Cost and token visibility** — declined on the planning map; reopens only on autopilot or explicit demand.
- **Concurrent-session resource limits** — no governor; the operator owns the machine, the rate limits, and the wallet.
- **The network and environment sandbox** — the operator owns sandboxing; a later effort, not a ticket here.
- **Redesigning the wayfinder method** — the harness extends the markdown adapter by one non-resolving heading and nothing more.
- **Charting as a harness capability** — the user's own `/wayfinder` flow in a shell; the harness only notices the result.
- **Autopilot** — strictly local, non-default, and not built here; the named reopen trigger for caps, clean-tree-at-spawn, machine stuck-detection, and cost.

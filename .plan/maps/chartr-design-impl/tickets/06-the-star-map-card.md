# The star-map card

## Question

The star-map renderer, reimplemented cleanly in TypeScript behind the narrow island seam — mount, receive the pushed model, emit selection — never decomposed into components (ADR 0010). Deterministic layout seeded from ticket data, the six base states, camera easing, with tuned constants (easing, zoom coupling, parallax, dpr) cribbed from the wayfinder-maps renderer against feel-drift; `starmap-design.md` and the planning map's prototype assets are the open references. The map is summoned as a floating card over the terminal — edge handle, `M`, Esc — never toggled by switching spaces or maps; the operator can instead dock it as the terminal-priority split (the terminal holds its pixel width; the map absorbs resize slack — planning ticket 08's amendment). An all-open map renders as-is.

Done when: fixture maps render with the six states; island-seam tests prove determinism (same data, same positions) and that model pushes change overlays but never move a star; summon/dismiss works by handle, key, and Esc; the split toggle docks the map without reflowing the terminal on window resize; clicking a star emits selection.

## Answer

The star-map renderer is reimplemented cleanly in TypeScript behind the narrow island seam — mount, receive the pushed model, emit selection — never decomposed into components (ADR 0010). The Svelte chrome hosts it but never reaches inside. Layout:

- **`web/src/lib/starmap/layout.ts`** — the deterministic layout as a pure function: mulberry32-seeded, rank-biased, force-relaxed with the constants cribbed verbatim from the wayfinder-maps renderer against feel-drift (`ringR = 130 + 165·rank`, `REP/SPRING/REST/RADIAL`, 420 relaxation steps). It depends only on ticket numbers and their `blocked_by` edges — never status — and seeds in ascending `num` order, so the same map lays out the same way regardless of the order tickets arrive in the snapshot. `structureSignature` is the identity the renderer keeps positions across; `edgesOf` drops edges that dangle onto absent tickets, so a malformed map still lays out (adoption is never gated on lint).
- **`web/src/lib/starmap/theme.ts`** — the six base states. The status derived from `.plan/` (ADR 0004) maps onto the viewer's ported palette: `open`+`frontier` → the bright takeable star, `open`+not-frontier → the small dim blocked one, plus `claimed`, `resolved`, `out_of_scope`, and `proposed` — chartr's one added base star (a warm, sealed body: work landed, the gate has not yet blessed it). Status is the whole star; ticket type rides only in the label. Session liveness (working/quiet/dead) and the review grammar (moons, beacon) are a strictly-additive overlay a later ticket layers on — this palette is only the base star.
- **`web/src/lib/starmap/starmap.ts`** — the imperative canvas island. `setModel` keeps every star's position when the map's structure is unchanged and recomputes layout only when tickets or edges change, so a pure status push changes a star's appearance but never its position. Camera pan/zoom with the ported easing (lerp 0.28, `exp(-Δ·0.0016)` zoom), a parallax starfield, flow-particle edges that ignite as blockers resolve, greedy label declutter, idle bob, and click-to-select with pane-aware camera easing (the star settles left-of-centre, leaving room for ticket 07's pane). It degrades gracefully with no 2D context — model, layout, and selection all work headless, drawing nothing — which is exactly what lets the seam be tested without a browser canvas.
- **`web/src/lib/StarMap.svelte` + `MapCard.svelte`** — the island wrapper (feeds each control-socket push to the island, lifts selection back out) and the card chrome: an optional map switcher, the dock/float toggle, dismiss, and a left-border resize grip. **`SpacePane.svelte`** summons the card over the terminal — edge handle, `M`, `Esc` — and docks it as the terminal-priority split. Visibility changes only on those explicit acts; switching spaces or focusing a different map never opens or closes it, so this state lives on the pane (persisting across space switches), not per-map.
- **`web/`** — vitest + jsdom stood up as the one frontend test point (the island seam), with a `test` script and a headless-canvas stub.

Against Done-when: `web/src/lib/starmap/starmap.test.ts` exercises the seam and pins the two binding guarantees. **Determinism** — the same fixture (and any ticket order) computes identical positions, and mixing in resolved/claimed/out-of-scope statuses leaves them identical, proving status is not a layout input. **Zero movement** — a five-ticket implementation fixture is walked through the whole lifecycle (claimed → proposed → resolved-igniting-the-dependent → out_of_scope) as a sequence of status-only pushes, and every star holds its exact starting position; a structural change (dropping a ticket) is the one path that relays out, and re-pushing the original map restores the original positions byte-for-byte. **Six states** — one push carrying all six renders every star. **Selection** — a click on a star emits its number and a click on empty space emits deselection; a stale selection is dropped when its ticket leaves the map. `svelte-check` (0 errors, 0 warnings), the Vite build, `vitest` (8 passing), `go vet`, and `go test` all pass.

Three decisions for review to weigh:

- **The card is resizable from its left border in both modes.** Docked, that border is the split divider: dragging it moves the map's edge and the terminal's frozen width follows, clamped so neither pane collapses. Floating, the card grows leftward while its right edge stays pinned. This extends planning ticket 08's "terminal holds its width" from a fixed 60/40 default to an operator-set split — the frozen-width, no-reflow-on-window-resize guarantee still holds; the operator simply chooses where the freeze lands. The grip is a pointer affordance (`svelte-ignore`d for a11y): keyboard users get the sensible dock/float defaults.
- **`M` and `Esc` are suppressed while the terminal owns the keyboard.** The shell's PTY takes every raw keystroke, so a global `M` would both type into the shell and summon the map. The keys fire only when focus is on the chrome (detected via `activeElement` / the terminal island); the always-live edge handle is the summon path when the shell is focused. This keeps the keyboard binding without hijacking terminal input, at the cost of `M` not working from inside a focused shell.
- **`proposed` is rendered as a sixth base star, not a session overlay.** The chartr model carries `proposed` as a derived status (ADR 0004), so it needs a base appearance now; the docked-moon/agent-review grammar from prototype 09 is the additive session overlay of a later ticket, which will layer on top of this base without moving anything.

Scope notes for review: the canvas *feel* is confirmed by eye in a browser (starmap-design.md, Open risk — it cannot be verified from a headless session), consistent with the prior UI tickets; the seam tests cover the determinism, no-movement, and selection guarantees the design record protects. The ticket pane the selection opens, the action-station badge on the handle, and the session/review overlays are named later tickets.

Review payload should carry this Done-when and the spec by assembly (spec, Prompts and payload).
